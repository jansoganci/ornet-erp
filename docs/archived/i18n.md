# i18n Kuralları (Dil ve Metinler)

Hiçbir metin hardcode Türkçe veya İngilizce yazılmaz. Tüm metinler çeviri anahtarları ile kullanılır; dil değişimi ve yeni dil ekleme kolay olur.

---

## 1. Kararlar

| Karar | Değer | Açıklama |
|-------|--------|----------|
| **Varsayılan dil** | `tr` (Türkçe) | Uygulama açıldığında kullanılan dil. |
| **Desteklenen diller (MVP)** | `['tr']` | Sadece Türkçe; yapı İngilizce’ye açık. |
| **Çeviri kütüphanesi** | `react-i18next` + `i18next` | Kurulum ve kullanım standardı. |
| **Çeviri dosyaları** | `src/locales/{locale}.json` veya namespace: `src/locales/{locale}/{namespace}.json` | Namespace: `common`, `customers`, `finance`, `auth`, `errors` vb. |
| **Tarih/sayı formatı** | `Intl` + locale | TR: 1.234,56 / EN: 1,234.56; para birimi ayrı alan (TRY, USD). |

---

## 2. Dosya Yapısı

```
src/
  locales/
    tr/
      common.json    # Genel: butonlar, etiketler, sayfalama
      auth.json      # Giriş, çıkış, şifre
      customers.json # Müşteri listesi, kart, form
      finance.json   # Kasa, ödeme, fatura
      service.json   # Servis/montaj formu, işler
      errors.json    # Hata mesajları
    en/
      common.json
      auth.json
      ...
```

MVP’de sadece `tr/` dolu olabilir; `en/` ileride eklenir.

---

## 3. Anahtar İsimlendirme (Key Convention)

- **Küçük harf, nokta ile hiyerarşi:** `customers.list.title`, `auth.login.submit`
- **Namespace kullan:** `t('customers:list.title')` veya `t('customers.list.title')` (i18next config’e göre)
- **Örnekler:**

| Anahtar | Örnek değer (tr) |
|---------|-------------------|
| `common.save` | Kaydet |
| `common.cancel` | İptal |
| `common.delete` | Sil |
| `auth.login.title` | Giriş Yap |
| `customers.list.title` | Müşteriler |
| `customers.detail.address` | Adres |
| `finance.accounts.title` | Kasalar |
| `service.form.type.service` | Servis |
| `service.form.type.installation` | Montaj |
| `errors.required` | Bu alan zorunludur |

---

## 4. Kullanım Kuralları

1. **Component içinde:** `const { t } = useTranslation('customers');` sonra `t('list.title')`
2. **Namespace belirt:** Her sayfa/ bileşen kendi namespace’ini kullanır; karışıklık olmaz.
3. **Değişken (placeholder):** `t('customers.welcome', { name: customerName })` — JSON’da `"welcome": "Hoş geldin, {{name}}"` gibi.
4. **Çoğul:** i18next pluralization kullanılabilir: `t('items.count', { count: n })` → "0 öğe", "1 öğe", "5 öğe".

---

## 5. Tarih ve Para Birimi

- **Tarih:** `new Intl.DateTimeFormat(locale, { dateStyle: 'medium' }).format(date)` veya dayjs/luxon + locale.
- **Sayı:** `new Intl.NumberFormat(locale).format(value)`
- **Para:** `new Intl.NumberFormat(locale, { style: 'currency', currency: 'TRY' })` — para birimi ekranda ayrı seçilebilir (TRY, USD).

Bu dosya tek referans; yeni ekran eklenirken hangi namespace’e hangi key ekleneceği buradan okunur.
